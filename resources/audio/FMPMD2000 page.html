What is FMPMD2000 / FMPMDE2000?

Software for playing song data for PC-98x1 series sound source drivers "FMP" and "PMD98" on Windows (FMPMD2000 is Japanese version, FMPMDE2000 is English version).

To listen to a song, you need WinFMP main unit (WinFMP.dll), PMDWin main unit (PMDWin.dll) and a player (of course, song data as well).

The current latest version is ver0.19 for WinFMP.dll, ver0.40 for PMDWin.dll, and ver0.31 for FMPMD2000 / FMPMDE2000.


In WinFMP.dll / PMDWin.dll, cisc's FM sound source engine is partially improved to emulate OPNA.

What is WatchFileUpdate?

It is a program that passes the update file to a specific executable file and executes it when the time stamp of the file is updated.

It may not come as a pin, but if you set it as shown in the figure, for example, the following processing will be performed automatically.

  1. When test.mml is saved in the editor, "c: \ bin \ mc.exe /vc:\data\test.mml" is executed.
  2. When the compiler generates test.m, it executes "c: \ bin \ fmpmd.exe c: \ data \ test.m".

I think it will come in handy for creating song data in a Windows environment.

Relationship between each program

  • WinFMP.dll

    • A DLL that generates PCM data during performance from FMP song data. It is close to the DOS version of FMP.com + PPZ8.com. It is designed to be intelligent and as least machine-dependent as possible.

    The reason why the DLL does not have the function to output sound is to reduce model dependency and because I am not good at creating window apps with VC ++ (PMDWin.dll, WinFMP.dll are VC ++). The second and third works).


  • PMDWin.dll

    • A DLL that generates PCM data during performance from PMD98 song data. It is close to the DOS version of PMDPPZ.com + PPZ8.com + P86drv.com + PDR.com + PPSDRV.com. It is designed to be as controllable as WinFMP.dll as much as possible.


  • FMPMD.exe

    • Controls WinFMP.dll and PMDWin.dll to play songs. It also provides a UI and visuals (spectrum analyzer, level meter, etc.). Corresponds to the DOS version of the player (play.com, pmp.com, etc.).

    It is also possible to create an original player by writing a program that controls the above DLL.


The reason why FMP is WinFMP.dll and PMD98 is PMDWin.dll is that when I first made the Windows version of PMD98, I chose PMDWin to be consistent with PMD98, PMD88, PMDTOWNS, ... etc. However, FMPWin was so bad that I chose WinFMP.


Sound source type

A brief description of the sound sources supported by FMP and PMD98.

  • FM sound source

    • By connecting four sine wave oscillators, the sound is modulated and the timbre is synthesized. You can choose from 8 types of connection methods. This is the most important sound source for FMP and PMD98.


  • PSG (SSG) sound source

    • A sound source that can play square waves or noise (random waves). Unlike FM sound sources, the waveform cannot be changed, but when used properly, it can express a unique and transparent tone.


  • Rhythm sound source

    • You can generate 6 types of sounds: bass drum, snare drum, hi-hat, cymbal, tom, and rim shot. Cheap tones invite tears (^^;


  • ADPCM sound source

    • It is a type of sampling that digitizes the sound captured from a microphone or external input and plays it, but it has the advantage that the data size is smaller than the PCM sound source described later because it is compressed using the difference from the previous data.


  • PCM sound source

    • Similar to the ADPCM sound source, it is a sound source that reproduces the data captured by sampling, but because it does not compress, the data size is large but the sound quality is excellent.


About the sound source chip

  • OPN (YM-2203)

    • FM sound source x 3 and SSG x 3 (both monaural) can be played. It is installed as standard on PC-9801 series after PC-9801DA, 98 compatible machines of Epson after PC-386GE, GS, etc., but it was omitted in the Windows era.


  • OPNA (YM-2608)

    • FM sound source x 6, SSG x 3, rhythm x 6 (stereo except SSG) can be played. ADPCM x 1 (stereo) can also be played if an external RAM is connected. It is possible to play ADPCM on the Speak board and Otomi-chan (both are the names of the sound source boards), but the 86 board released by NEC is equipped with PCM x 1 (stereo), so ADPCM has been omitted. It is installed as standard on the PC-9821-A Mate and Multi series.


About FMP

  • FMP.COM

    • FMP for OPN, Speak board, 86 board. Maximum FM x 6, SSG x 3, rhythm x 6, ADPCM x 1 (stereo) can be played. You can use PPZ8, which will be described later, to play PCM x 3 instead of SSG.


About the types of PMD98

  • PMD.COM

    • PMD for OPN. FM x 3 and SSG x 3 (monaural) can be played.


  • PMDB2.COM

    • PMD for Speakboard and Otomi-chan. FM x 6, SSG x 3, rhythm x 6, ADPCM x 1 (stereo) can be played.


  • PMD86.COM

    • PMD for 86 board. By using P86DRV, which will be described later, you can play almost the same as PMDB2.


  • PMDPPZ.COM

    • PMD for 86 board. By using PPZ8, which will be described later, you can combine 8 parts with a PCM part that has only 1 part and play it.


About auxiliary programs

  • PDR, PPSDRV

    • This is a program for playing sampling data by force using the SSG part. It only works when used in pairs. Use .PPS as the sampling data. It can be used with all types of PMD98.


  • P86DRV

    • This is an auxiliary program for playing PCM on PMD86. Use .P86 as the sampling data.


  • PPZ8

    • Combines 8 sounds of PCM in combination with FMP or PMDPPZ (FMP supports only 3 sounds). Cannot be used with P86DRV. Use .PVI or .PZI as the sampling data.


About song data and sampling data extensions

.OPI Song data (OPN) for FMP.
.OVI Song data (OPNA) for FMP.
.OZI Song data for FMP (using PPZ8, OPN or OPNA).
.M It is an extension for PMD (OPN), but it seems that some song data of PMDB2, PMD86, PMDPPZ also have this extension.
.M2 Song data for PMDB2 or PMD86.
.MZ Song data for PMDPPZ.
.PVI Originally it was FMP sampling data (ADPCM), but it can also be used as PPZ8 part (FMP, PMDPPZ) and PMDB2 sampling data.
.PZI Sampling data (PCM) for PPZ8. It can be used with PMDPPZ.
.PPS Sampling data (PCM) for PDR and PPSDRV.
.PPC Sampling data (ADPCM) for PMDB2.
.P86 Sampling data (PCM) for PMD86.

FMPMD2000 supports all song data (PMDB2, PPSDRV, P86DRV, PPZ8) of FMP and PMD98.



About data creation

FMP or PMD98 song data is created by compiling a language for writing music called MML (Music Macro Language) with a compiler. However, the language specifications differ between FMP and PMD98. See the manual that came with the original FMP or PMD98.

Currently, there is no GUI-based creation software for Windows, so you can create it by the following method.


  1. Created with the PC-98x1 series.

    2. This is the most reliable method, but of course you need to have a real machine and a sound source.


  2. Create on the emulator.

    3. Emulators such as Anex86, Virtual98, T98, Neko Project II can be used. However, the PPZ8 emulation is easy to skip.


  3. Run mc.exe or fmc.exe (corresponding version) directly at the Windows DOS prompt.

    4. When using fmc.exe on an AT compatible machine, only the version (3.16z) excluding the PC-98x1 machine-dependent part can be used. If you use version 3.16z or earlier, the worst OS will be down.

    Never use the / P, / S options of mc.exe on AT compatibles. Also, never run the evaluation compiler mch.exe on the Window. Worst case OS goes down.

    If you are using the PC-98x1 series (with sound source), you can develop at the DOS prompt, but if you use PPZ8, set Windows so that you do not use the same PCM device.

    The version (3.16z) of the FMP compiler fmc.exe excluding the PC-98x1 machine-dependent part has been released. Guu-san, thank you for your hard work (2001/09/16)

    PMD98 compiler mc.exe is now free software. Thank you KAJA (2000/06/17)

    By using WatchFileUpdate (my work), it is now possible to automatically compile and play in conjunction with saving MML files (2001/02/17).
